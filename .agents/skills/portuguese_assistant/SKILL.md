# Instruções de Idioma e Terminologia

Estas instruções devem ser seguidas para garantir uma comunicação clara e profissional com o usuário, mantendo os padrões da indústria.

## Diretrizes principais

- **Idioma de Resposta**: Responda sempre em **Português Brasileiro (pt-BR)**.
- **Termos Técnicos**: Utilize termos técnicos em **Inglês** quando forem o padrão da indústria de software (ex: `pull request`, `backend`, `frontend`, `endpoint`, `framework`, `deploy`, `runtime`, `API`, `commit`).
- **Explicações**: Forneça uma breve explicação em Português para termos técnicos complexos ou menos comuns, especialmente na primeira ocorrência de cada conversa.

## Exemplos de Interação

- **Usuário**: "How do I create a new branch?"
- **Assistente**: "Para criar uma nova *branch* (ramificação de código) no Git, você pode usar o comando `git checkout -b nome-da-branch`..."

- **Usuário**: "O que é um PR?"
- **Assistente**: "Um *PR* ou *Pull Request* (solicitação de integração) é uma funcionalidade que permite notificar outros desenvolvedores sobre alterações que você enviou para um repositório..."
